# Instructions

## Mục đích
- Tìm hiểu và so sánh các định dạng/công cụ thiết kế API phổ biến (OpenAPI, API Blueprint, RAML, TypeSpec, TypeAPI).
- Đánh giá ưu nhược điểm của OpenAPI so với các đối thủ.

## Yêu cầu
- Nắm vững cú pháp cơ bản của từng định dạng thông qua các file đi kèm.
- Hiểu được hệ sinh thái và ứng dụng thực tế của chúng trong các quy mô dự án khác nhau.
- Chạy được ví dụ demo trong các thư mục tương ứng theo hướng dẫn ở mỗi `README.md`.

## Phương pháp so sánh
- Đánh giá dựa trên các tiêu chí: Định dạng (Format), Hệ sinh thái (Ecosystem), Khả năng dễ đọc (Readability), Khả năng sinh code (Code Generation), và Mức độ phù hợp.

## Dàn ý chung
1. Giới thiệu tổng quan về OpenAPI và vai trò của tài liệu hóa API.
2. Trình bày các định dạng/công cụ thay thế.
3. Bảng so sánh chi tiết.
4. Demo thực tế qua các ví dụ tham khảo.
